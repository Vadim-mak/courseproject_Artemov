# Техническое задание
## Информационная система туристического агентства

**СКФУ, 2026 г. | Траектория В: Мобильная разработка**

---

## 1. Общие сведения

**Наименование системы:** Информационная система туристического агентства

**Назначение:** Мобильное приложение и серверная часть для автоматизации процессов туристического агентства: каталога туров, онлайн-бронирования и управления заявками.

**Архитектура:** PCMEF (Presentation — Control — Mediator — Entity — Foundation)

---

## 2. Требования к системе

### 2.1 Функциональные требования

**Для туриста (ROLE_USER):**
- Регистрация и аутентификация по email/паролю
- Просмотр каталога туров с пагинацией
- Поиск туров по ключевому слову, стране, диапазону цен
- Просмотр детальной информации о туре
- Бронирование тура (выбор количества туристов)
- Просмотр истории своих бронирований
- Отмена бронирования
- Оставить отзыв о туре (оценка 1–5, комментарий)
- Редактирование профиля (имя, телефон)

**Для администратора (ROLE_ADMIN):**
- Все права туриста
- Создание, редактирование и деактивация туров
- Управление справочником направлений (стран)
- Просмотр всех бронирований
- Подтверждение бронирований
- Просмотр списка пользователей

### 2.2 Требования к архитектуре (PCMEF)

| Слой | Реализация | Ответственность |
|---|---|---|
| Presentation | React Native (Expo) | Экраны, компоненты, навигация |
| Control | Spring Boot @RestController | HTTP-маршрутизация, валидация запросов |
| Mediator | @Service + интерфейсы IxxxService | Бизнес-логика, транзакции |
| Entity | JPA @Entity с бизнес-методами | Состояние, доменные правила |
| Foundation | Spring Data JpaRepository | Доступ к данным, JPQL-запросы |

**Правило зависимостей:** Control → Mediator → Entity ← Foundation. Прямые обращения Control → Foundation запрещены.

### 2.3 Требования к безопасности

- Аутентификация: JWT (jjwt 0.12), срок действия токена — 1 час
- Хранение токена: AsyncStorage (мобильный клиент)
- Хеширование паролей: BCrypt (strength=10)
- Авторизация: роли ROLE_USER, ROLE_ADMIN
- Защита: Spring Security stateless, CORS для мобильных клиентов
- Сессии на сервере отсутствуют (SessionCreationPolicy.STATELESS)

### 2.4 Требования к REST API

- Минимум 8 эндпоинтов (реализовано 27)
- Документация OpenAPI: http://localhost:8080/swagger-ui.html
- Версионирование: /api/...
- Стандартные HTTP-статусы: 200, 201, 204, 400, 401, 403, 404, 409, 500
- Пагинация: Spring Pageable (page, size, sort)
- Единый формат ошибки: `ApiError(timestamp, status, error, message, details)`

### 2.5 Требования к надёжности

- При сетевой ошибке мобильный клиент отображает Alert с текстом ошибки
- Токен с истёкшим сроком автоматически удаляется из AsyncStorage
- Транзакционность: @Transactional на всех методах Mediator, изменяющих данные
- Ссылочная целостность: `ON DELETE CASCADE` для user_roles

## 4. Требования к развёртыванию

### Локальный запуск

```bash
# PostgreSQL
docker-compose up -d postgres

# Backend
cd backend && mvn spring-boot:run

# Mobile
cd mobile && npm install && npm start
```

### Docker (всё сразу)

```bash
docker-compose up -d
```

### Системные требования сервера

- JDK 17+
- PostgreSQL 14+
- RAM: минимум 512 МБ
- Порт: 8080 (backend), 5432 (PostgreSQL)
